## рҹ“– Overview

This project implements a **4Г—4 unsigned binary multiplier** entirely at the **gate/structural level** in Verilog вҖ” no `*` operator is used anywhere. The multiplier is built layer by layer:

- A **Full Adder** is designed using basic Boolean equations (`assign`).
- Four Full Adders are chained to form a **4-bit Ripple Carry Adder (RCA)**.
- The RCA is then reused **three times** inside the multiplier to add shifted partial products together, exactly the way multiplication is done by hand (long multiplication in binary), producing an **8-bit product** from two 4-bit unsigned inputs.

This bottom-up design approach (Full Adder вҶ’ RCA вҶ’ Multiplier) is a classic digital design technique used to teach and build **array multipliers**, which are widely used in ALUs, DSPs, and processor datapaths.

## рҹ§© Module-by-Module Explanation

### 1пёҸвғЈ Full Adder (`fulladder.v`)

> рҹ“„ Source code: see [`fulladder.v`](./fulladder.v) in this repo.

**Ports:**
| Port | Direction | Width | Description |
|---|---|---|---|
| `a, b` | input | 1-bit | Operand bits to add |
| `cin` | input | 1-bit | Carry-in from previous stage |
| `s` | output | 1-bit | Sum bit |
| `c` | output | 1-bit | Carry-out |

**Logic:**
- `s = a вҠ• b вҠ• cin` вҖ” standard XOR sum equation
- `c = (aВ·b) + (bВ·cin) + (aВ·cin)` вҖ” majority function, produces carry-out

This is a purely **combinational**, gate-level description of the classic 1-bit full adder.

---

### 2пёҸвғЈ Ripple Carry Adder (`rca.v`)

> рҹ“„ Source code: see [`rca.v`](./rca.v) in this repo.

**Ports:**
| Port | Direction | Width | Description |
|---|---|---|---|
| `a, b` | input | 4-bit | 4-bit operands |
| `cin` | input | 1-bit | Initial carry-in |
| `s` | output | 4-bit | 4-bit sum |
| `c` | output | 4-bit | Per-stage carry-outs (`c[0]..c[3]`) |

**Working:** Four `fulladder` instances are **chained** вҖ” the carry-out of each stage feeds the carry-in of the next stage (`c[0]вҶ’f2`, `c[1]вҶ’f3`, `c[2]вҶ’f4`), forming a classic **ripple carry adder**. `c[3]` is the final carry-out of the 4-bit addition.

> Note: in this design, the `rca` module's `c` output is used flexibly by the multiplier вҖ” it is wired to intermediate carry/sum bits rather than just a single final carry, since the multiplier reuses it as a general 4-bit adder block.

---

### 3пёҸвғЈ Array Multiplier (`multiplication4x4.v`)

> рҹ“„ Source code: see [`multiplication4x4.v`](./multiplication4x4.v) in this repo.

**Ports:**
| Port | Direction | Width | Description |
|---|---|---|---|
| `a, b` | input | 4-bit each | Multiplicand and Multiplier |
| `out` | output | 8-bit | Final product (a Г— b) |

**Internal signals:**
- `w[14:0]` вҖ” holds all 15 partial products (AND of `a[i]` and `b[j]`) other than `out[0]`
- `w1, w2, w3 [4:0]` вҖ” intermediate sum/carry results from each RCA stage
- `w5, w6, w7 [2:0]` вҖ” intermediate carry chains reused between RCA stages

**Step-by-step working:**

1. **Partial Product Generation (AND array):**
   Each bit of `a` is ANDed with each bit of `b`, generating 16 partial product bits total (`a[i] & b[j]`), forming a 4Г—4 grid вҖ” exactly like manual binary long multiplication.

2. **Row 0 (weight 2вҒ°):** `out[0] = a0В·b0` (LSB вҖ” needs no addition, direct output)

3. **First RCA (`r1`):** Adds Row-0 partial products (shifted by 1) with Row-1 partial products (`b1В·a[3:0]`), producing an intermediate sum `w1` and carry chain `w5`. `out[1]` is taken directly from this stage (`w1[0]`).

4. **Second RCA (`r2`):** Adds the shifted result of `r1` (`w1[4:1]`) with Row-2 partial products (`b2В·a[3:0]`), producing `w2` and carry chain `w6`. `out[2] = w2[0]`.

5. **Third RCA (`r3`):** Adds the shifted result of `r2` (`w2[4:1]`) with Row-3 partial products (`b3В·a[3:0]`), producing the final 5-bit result `w3[4:0]`, which directly becomes `out[7:3]` вҖ” the most significant bits of the product.

This is a textbook **shift-add array multiplier**: each row of partial products is added to the running sum, shifted one position to the left, using cascaded ripple-carry adders вҖ” just like doing multiplication by hand in binary.

## рҹ§  How the Multiplier Works (Theory)

Binary multiplication of two 4-bit numbers works exactly like decimal long multiplication:

```
     7     6     5     4     3     2     1     0
                            a3    a2    a1    a0
  Г—                         b3    b2    b1    b0
----------------------------------------------------
                          a3b0  a2b0  a1b0  a0b0   <- Row0 (Г—b0), shift 0
                    a3b1  a2b1  a1b1  a0b1         <- Row1 (Г—b1), shift 1
              a3b2  a2b2  a1b2  a0b2               <- Row2 (Г—b2), shift 2
        a3b3  a2b3  a1b3  a0b3                     <- Row3 (Г—b3), shift 3
----------------------------------------------------
    p7    p6    p5    p4    p3    p2    p1    p0   <- 8-bit product
```

Each row is generated using **AND gates** (a partial product), and the rows are summed together with progressive left-shifts using **Ripple Carry Adders**, exactly as implemented in `multiplication4x4.v`. This is known as an **Array Multiplier** вҖ” a fully combinational, gate-efficient multiplier architecture.

## вҡҷпёҸ How to Run (Icarus Verilog + GTKWave)

### Prerequisites

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install iverilog gtkwave
```

**macOS (Homebrew):**
```bash
brew install icarus-verilog
brew install --cask gtkwave
```

**Windows:**
1. Download and install **Icarus Verilog** from the official Windows installer: [http://bleyer.org/icarus/](http://bleyer.org/icarus/) (pick the latest `iverilog-vX.X-setup.exe`). During setup, make sure to check the option to **add Icarus Verilog to PATH**, or add it manually (`C:\iverilog\bin`) to your System Environment Variables.
2. Download and install **GTKWave** вҖ” it's bundled with the Icarus Verilog Windows installer above, or can be installed separately from [https://gtkwave.sourceforge.net/](https://gtkwave.sourceforge.net/).
3. Verify installation by opening **Command Prompt** or **PowerShell** and running:
   ```powershell
   iverilog -V
   gtkwave --version
   ```
4. (Alternative) Windows users can also install both tools via **Chocolatey**:
   ```powershell
   choco install icarus-verilog
   ```
5. (Alternative) Or use **WSL (Windows Subsystem for Linux)** and follow the Ubuntu/Debian commands above inside your WSL terminal.

### Run the Full Adder simulation
```bash
iverilog -o fulladder_tb.out fulladder_tb.v fulladder.v
vvp fulladder_tb.out
gtkwave dump.vcd
```

### Run the Ripple Carry Adder simulation
```bash
iverilog -o rca_tb.out rca_tb.v rca.v fulladder.v
vvp rca_tb.out
gtkwave dump.vcd
```

### Run the 4Г—4 Multiplier simulation
```bash
iverilog -o multiplication_tb.out multiplication_tb.v multiplication4x4.v rca.v fulladder.v
vvp multiplication_tb.out
gtkwave dump.vcd
```

> вҡ пёҸ Always compile the **dependency modules first** (`fulladder.v`, then `rca.v`) before the top-level module, since `rca` instantiates `fulladder`, and `multiplication` instantiates `rca`.

## рҹ§® How to Run (Xilinx Vivado)

1. Open **Vivado** вҶ’ `File вҶ’ New Project` вҶ’ select **RTL Project**.
2. Add source files: `fulladder.v`, `rca.v`, `multiplication4x4.v`.
3. Add simulation sources: `multiplication_tb.v` (and other testbenches as needed).
4. Set `multiplication` (or the relevant module) as the **Top Module**.
5. Run **Behavioral Simulation**:
   `Flow Navigator вҶ’ Simulation вҶ’ Run Simulation вҶ’ Run Behavioral Simulation`
6. To view the **RTL Schematic**:
   `Flow Navigator вҶ’ RTL Analysis вҶ’ Open Elaborated Design вҶ’ Schematic`
7. Export the schematic as PNG: `File вҶ’ Export вҶ’ Export Schematic` or take a screenshot for `schematic.png`.
